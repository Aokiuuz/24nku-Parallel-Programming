随着云端服务并发度不断提高，协程（coroutine）技术应运而生。这是一种比线程更加轻量级的并发编程模型，你可以把它通俗地理解为“可暂停和恢复的函数”。它具有极轻量级、协作式调度、异步编程简单等特点，非常适合于I/O密集型。主流编程语言都提供了对协程支持，例如C++20标准引入的协程底层机制、Go语言的Goroutine、Python的 asyncio、C#的async/await以及Java 21引入的虚拟线程等。
请对协程技术进行调研，内容包括（但不限于）：并发执行机制为什么一步步演进到协程；协程技术介绍、分析，包括其底层机制、应用范式等；编程语言对协程支持的介绍和对比；协程编程实验，包括与多线程的性能对比等；编程语言（编译器）、运行时关键、操作系统是如何协同，从底层实现协程机制的。
