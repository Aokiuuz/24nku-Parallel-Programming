Lab02 SIMD 口令猜测当前记录

1. 当前 mycode/guess 的实现范围

当前代码主要实现了 MD5 哈希阶段的 ARM NEON 4 路 SIMD 并行：

- mycode/guess/md5.cpp 中实现了 MD5Hash_simdversion(string input[4], uint32x4_t state[4])。
- 该函数一次传入 4 个口令，使用 uint32x4_t 的 4 个 lane 分别计算 4 个口令的 MD5。
- F/G/H/I、FF/GG/HH/II、循环左移、加法、异或、与或非等 MD5 轮函数被改成 NEON 向量运算。
- mycode/guess/main.cpp 已回退为直接测试 SIMD 的单一路径版本，每 4 个 q.guesses 直接调用一次 MD5Hash_simdversion。
- 不足 4 个的尾部口令仍用串行 MD5Hash 处理。
- mycode/guess/main.cpp 当前不保留 --serial 开关；串行对照使用已有 data/test429.o 结果或另行切换串行版本测试。

尚未并行化的部分：

- PCFG 训练仍是串行。
- PriorityQueue 维护仍是串行。
- guessing.cpp 中 PriorityQueue::Generate 的候选口令生成循环仍是串行。
- 当前没有实际使用 OpenMP、pthread、MPI 或 CUDA。

因此当前可以表述为：完成了基础 SIMD 要求中的 ARM NEON 4 路 MD5 并行哈希；PCFG 候选生成和模型训练仍为串行。

3. 加速比

Hash 阶段加速比：

2.86369 / 1.85658 = 1.5424544054

约为 1.54 倍。

Guess + Hash 阶段总时间：

- 串行：0.535787 + 2.86369 = 3.399477 seconds
- SIMD：0.55067 + 1.85658 = 2.40725 seconds

Guess + Hash 总体加速比：

3.399477 / 2.40725 = 1.4121827812

约为 1.41 倍。

说明：

- 这里的 Hash 阶段加速比更能体现 SIMD MD5 的效果，因为当前只并行化了哈希阶段。
- Guess time 主要来自 PCFG 候选口令生成，当前仍是串行，所以总体加速比低于 Hash 阶段加速比。

4. 2026-05-08 代码调整记录

基础版本 mycode/guess：

- main.cpp 已回退为接近原始框架的单一路径版本，用于直接测试 SIMD。
- 保留服务器训练数据路径 /guessdata/Rockyou-singleLined-full.txt。
- 哈希阶段在 q.guesses 达到批量阈值后，按 4 个口令一组调用 MD5Hash_simdversion。
- 尾部不足 4 个的口令仍调用串行 MD5Hash，避免越界。
- 保留 Guess time / Hash time / Train time 三行标准输出，方便 test.o 直接读取。

进阶版本 mycode/guess_advanced：

- md5.h 中将 MD5Hash 改为 const string & 参数，减少串行调用时的字符串复制。
- md5.h 中将 MD5Hash_simdversion 改为 const string input[4] 参数，允许 main.cpp 直接传 vector<string> 内部连续地址。
- main.cpp 的 HashGuessesSimd 不再构造 string pw_batch[4]，改为 MD5Hash_simdversion(&guesses[i], state)，减少每 4 个口令一次的字符串复制。
- main.cpp 增加 MD5Hash_simdversion 正确性测试，启动时验证 8 个常见口令的 4 路 SIMD lane 输出。
- md5.cpp 中将 SIMD 的 F/G/H/I、ROTATELEFT、FF/GG/HH/II 改为宏展开，减少 -O0 下大量函数调用开销。
- md5.cpp 中 ROTATELEFT_SIMD 改用 vshlq_n_u32 / vshrq_n_u32 常量移位，避免变量移位指令和额外向量构造。
- md5.cpp 新增 BuildSingleBlockWordsSimd fast path：长度不超过 55 字节的 4 个口令直接在栈上构造 16 个 MD5 message word，不再调用 StringProcess，也不再 new/delete paddedMessage。
- md5.cpp 保留 MD5Hash_simdversion_slow 作为长口令 fallback；如果 4 个长口令 padding 后块数不同，则退回 PackSerialHashes 用串行结果打包成 SIMD state，保证正确性。
- 当前进阶版目标是降低 SIMD 前后处理开销，并提升 -O0、-O1、-O2 下的 Hash 阶段加速比。
